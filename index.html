Preliminary Software Design

Model

Card Class

A card class will be instantiated with the following arguments: SUIT, RANK, VALUE. The suit will contain a string with the type of suit, the rank will contain a numerical value assigned to the face value of the card (ex: Jack = 11, 2 = 2, etc). The value of the card will be equivalent to the value in the game, 1-11, face cards = 10 and ace = 1 or 11.

Card Object Example: {suit: “hearts”, rank: 12 (Queen), value: 10}

Player Class

A player class will hold he following arguments: PLAYER ID, NAME, HAND, CHIPS. The player id will be used to identify a player in the background process while the name will be used for the purposes of user identification. The hand will contain the cards held by the player and the chips equal to the number of chips currently held by the player.

Player Object Example: {playerId: 1, name: “John Doe”, hand: [{card object}, {card object}, etc], chips = 500}

Table Class

The table class will manage much of the game, it will hold discarded cards, the deck, etc. The table class will hold the following information: DISCARD_PILE, PLAY_DECK, POT. The discard_pile will hold an array of discarded card objects, the play deck will hold cards that can still be used, the pot will total chips used in betting to be allotted to the winner at the end.

Table Object Example: {discard_pile: [{card object}, {card object}, etc], play_deck: [{card object}, {card object}, etc], pot = 1000}

Functions/Methods
  • cardFactory() – Generates the initial unshuffled deck of card objects to be used at the beginning of the game.
  • shuffleDeck() – Shuffles the cards and can be used at the end of the round to reshuffle the remaining cards in the playing deck.
  • deal() – Plays a card to a player from the play deck
  • bet() – A player can specify the amount desired to bet during a round
  • hit() – Gives a player a new card
  • pass() – finishes the turn for the player
  • endOfRound() – manages the game functions at the end of the round, discards all cards in play.

Project Game Description

The game I will be designing, and implementing is Blackjack. The game is played with a deck of cards. Players can place bets prior to being dealt cards. Each player, including the dealer receive 2 cards. Each card is valued at the number assigned to the card, if it is a face card, it is assigned the value 10. Ace is either counted as 11 or 1 depending on the players choice. The goal of the game is to get as close to 21 as possible without going over. If the player or dealer go over, it is called a bust. If the player is closer to 21 than the dealer, the player wins 1:1 the amount they bet. Otherwise, if the dealer wins, the player will lose the amount they bet.
